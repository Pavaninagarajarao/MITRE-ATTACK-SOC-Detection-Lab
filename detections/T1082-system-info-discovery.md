# Detection: System Information Discovery

**ATT&CK Technique:** T1082 - System Information Discovery
**Tactic:** Discovery
**Data Source:** Sysmon Event ID 1 (Process Creation)
**Severity:** Medium

## Description
Detects execution of native Windows utilities commonly used by adversaries to enumerate system information (OS version, hardware, hostname, domain) during the discovery phase of an attack.

## Splunk Query

    index=sysmon EventCode=1
    | where like(CommandLine, "%tasklist%") OR like(CommandLine, "%systeminfo%") OR like(CommandLine, "%wmic%")
    | table _time, Computer, User, ParentImage, Image, CommandLine
    | sort -_time

## True Positive Example
This query returned 68 matching events in the lab environment, including `systeminfo.exe`, `wmic`, and `tasklist` executions launched from `cmd.exe`. Cross-referencing the `User` and `ParentImage` fields confirmed these were generated by running Atomic Red Team's T1082 test, consistent with adversary discovery behavior.

## False Positive Considerations
IT asset management tools, helpdesk scripts, monitoring agents, and routine system audits frequently run `systeminfo`, `wmic`, and `tasklist` as part of normal operations. These commands are also commonly used by support staff for troubleshooting. Tune by whitelisting known ParentImage values for RMM tools, monitoring agents, or helpdesk software, and treat this as low-confidence on its own — correlate with other suspicious activity (e.g. unusual parent processes, off-hours execution, or unfamiliar user accounts) before escalating.

## Response Actions
1. Identify the parent process and user account that triggered the command
2. Check whether the activity aligns with scheduled IT/asset-management jobs
3. Look for this event alongside other discovery techniques to assess if it's part of a broader recon chain
4. Escalate if discovery activity is followed by lateral movement or credential access attempts
