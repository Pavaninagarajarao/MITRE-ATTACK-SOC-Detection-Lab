# Detection: Suspicious Encoded PowerShell Execution

**ATT&CK Technique:** T1059.001 - Command and Scripting Interpreter: PowerShell
**Tactic:** Execution
**Data Source:** Sysmon Event ID 1 (Process Creation)
**Severity:** High

## Description
Detects PowerShell processes launched with encoding flags commonly used to obfuscate malicious commands.

## Splunk Query

    index=sysmon EventCode=1 Image="*powershell.exe*"
    (CommandLine="*-enc*" OR CommandLine="*-EncodedCommand*" OR CommandLine="*hidden*")
    | table _time, Computer, User, ParentImage, CommandLine

## True Positive Example
Atomic Red Team T1059.001 Test 2 generates encoded PowerShell execution that triggers this rule reliably.

## False Positive Considerations
Some legitimate admin scripts and deployment tools use encoded PowerShell. Tune by whitelisting known ParentImage values like SCCM or your RMM tool.

## Response Actions
1. Identify the parent process and user account
2. Decode the base64 payload using CyberChef
3. Check for follow-on network connections (Sysmon Event ID 3)
4. Escalate if payload contains download cradles or known C2 patterns

## Tuning Query (Frequency Analysis)
Use this to identify the most common command lines and reduce noise before finalizing the detection:

    index=sysmon EventCode=1 Image="*powershell.exe*"
    | stats count by CommandLine, ParentImage, User
    | sort -count
